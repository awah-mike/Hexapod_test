# Hexapod_test

RL training pipeline for a 6-legged, 3-DOF-per-leg hexapod robot. Designed for
sim training in NVIDIA Isaac Lab, with eventual sim-to-real deployment.

## Robot

- **6 legs** labeled `FL`/`FR`/`ML`/`MR`/`BL`/`BR` (front/middle/back × left/right).
- **3 DOF per leg**: coxa (yaw), femur (lift), tibia (knee). Total **18 actuated joints**.
- **Bilateral symmetry** across the YZ plane. Not radial like Anymal.
- **Body frame**: +X = right side, -X = left side, +Y = forward, +Z = up.

## URDF design notes

The original Fusion 360 export had all 18 joints as `type="fixed"` (Rigid joints
in Fusion). `scripts/fix_urdf.py` converts them to `type="revolute"` with:

### Rotation axes
- **Coxa joints**: `(0, 0, ±1)` — vertical body axis. Sign flipped per side
  (`+1` for right, `-1` for left) so a positive command swings any leg forward.
- **Femur and tibia joints**: per-leg horizontal axes specified by the designer
  (see `THETA_DEG` in `fix_urdf.py`). For each right-side leg, axis = `(sin θ, -cos θ, 0)`
  where θ is measured CCW about +Z from -Y. Left side is mirrored across YZ.

### Joint limits
| Joint | Right (rad) | Left (rad) | Range |
|---|---|---|---|
| Coxa | [-0.7854, +0.7854] | [-0.7854, +0.7854] | ±45° |
| Femur | [0, +1.2217] | [-1.2217, 0] | 70°, asymmetric due to mirrored axes |
| Tibia | [0, +2.4435] | [-2.4435, 0] | 140°, asymmetric due to mirrored axes |

### Inertia
Mass per link type: coxa = 0.2 kg, femur = 1.0 kg, tibia = 0.7 kg (user-specified).
Inertia tensors are diagonal placeholders (1e-4, 1e-3, 7e-4 kg·m²) about the
center of mass. The CoM positions are inherited from Fusion's export (real CAD
geometry). Inertia magnitudes are estimates; revisit before sim-to-real.

## Workflow

### 1. Regenerate the URDF (only after editing `fix_urdf.py` or the source xacro)
```bash
python3 scripts/fix_urdf.py
```

### 2. Sanity-check at q=0
```bash
python3 scripts/visualize_urdf.py assets/robots/hexapod/urdf/hexapod.urdf hexapod_zero_pose.html
# Open hexapod_zero_pose.html in any browser. Drag to rotate, scroll to zoom.
```

### 3. Convert URDF to USD (run on a machine with Isaac Sim, e.g. Brev "Isaac Launchable")
```bash
# Inside the Isaac Sim Python env:
/workspace/isaaclab/_isaac_sim/python.sh scripts/convert_urdf_to_usd.py
# or via isaaclab.sh -p
```
Produces `assets/robots/hexapod/hexapod.usd`. Commit it to the repo.

### 4. Train (downstream, not in this repo yet)
The USD will be consumed by an Isaac Lab `ArticulationCfg`. The env class,
PPO config, and training scripts live in a separate package (TBD).
